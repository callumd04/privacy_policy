Advanced routing configuration

Vercel gives you a variety of ways to configure the routing of your website. For example, adding redirects, rewrites, or headers to your page. While you can use vercel.json to define these through JSON, you can also express your routing configuration as code using Vercel Edge Middleware.

Middleware runs before the Vercel Edge Network Cache, so you can configure routing exactly as your application needs. For example, this template uses middleware.js defined in the root of the repository to add additional security headers to the site: 

referrer-policy: origin-when-cross-origin
strict-transport-security: max-age=31536000; includeSubDomains; preload
x-content-type-options: nosniff
x-dns-prefetch-control: on
x-frame-options: DENY

Learn more about Vercel Edge Middleware.
